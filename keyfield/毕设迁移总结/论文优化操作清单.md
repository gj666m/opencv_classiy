### 2.1 新增推理速度对比实验

**问题：** 表 4-1 中 ShuffleNetV2 定位为"速度对照模型"，但全文没有任何推理速度或 FPS 数据。

**🔗 前端重构关联：** 前端模型对比功能（`/api/compare`）会自动记录每个模型的推理耗时（ms），可直接从浏览器界面截图或从后端日志提取数据用于论文表格，无需单独写测试脚本。

**操作步骤：**
1. 启动重构后的 Web 系统，使用"智能识别 → 模型对比"功能上传图片
2. 截取对比结果截图（含四宫格 + 推理耗时表格），用于论文插图
3. 将模型对比接口返回的推理耗时数据整理为论文表格：

   | 模型 | 参数量 | 权重大小 | 单张推理耗时(ms) | FPS | Accuracy |
   |------|-------:|--------:|-----------------:|----:|---------:|
   | MobileNetV3-Small | 2.5M | 17.67MB | | | 92.38% |
   | ResNet18 | 11.7M | 128.06MB | | | 93.57% |
   | EfficientNet-B0 | 5.3M | 46.39MB | | | **94.47%** |
   | ShuffleNetV2 | 2.3M | 14.70MB | | | 93.39% |

4. 将该表格插入论文 6.2 节末尾
5. 在文中分析：EfficientNet-B0 虽然精度最高，但在资源受限场景下 ShuffleNetV2/MobileNetV3-Small 的推理速度优势更明显

### 2.2 补充 OpenCV 预处理策略对比实验

**问题：** 论文题目强调 OpenCV，但只使用了高斯去噪 + CLAHE 两种操作。实际代码中 `preprocess.py` 还实现了中值滤波、双边滤波、边缘检测（Canny/Sobel/Laplacian）、亮度调整、对比度调整等功能，但论文一个字没提。

**🔗 前端重构关联：** 前端 OpenCV 演示页已实现管线可视化（原图→去噪→CLAHE→最终输入），可从中截取效果图直接用于论文插图。但不同预处理策略的对比实验仍需单独跑训练。

**操作步骤：**
1. 利用已有 `preprocess.py` 中的多种方法，对同一组样本图片生成预处理对比图：
   - 原图 vs 高斯去噪 vs 中值去噪 vs 双边滤波
   - 原图 vs CLAHE(2.0) vs CLAHE(4.0) vs 直方图均衡化
   - 原图 vs 亮度增强 vs 对比度增强 vs 边缘检测
2. 使用 EfficientNet-B0，分别在不同预处理配置下训练几个 epoch（如 10 epoch），记录验证准确率，形成对比表：

   | 预处理策略 | Val Accuracy (10 epoch) | 说明 |
   |-----------|:----------------------:|------|
   | 无预处理 | | 基线 |
   | 高斯去噪 | | |
   | 高斯去噪 + CLAHE | | 当前方案 |
   | 中值去噪 + CLAHE | | |
   | 双边滤波 + CLAHE | | |

3. 在论文 3.3 节或新增 6.X 节"OpenCV 预处理策略对比"中：
   - 插入预处理效果对比图（3-4 组）
   - 插入对比实验表格
   - 分析为什么最终选择高斯去噪 + CLAHE 的组合
4. 这项工作既增加了 OpenCV 工作量，也为论文增加了独立的实验分析
