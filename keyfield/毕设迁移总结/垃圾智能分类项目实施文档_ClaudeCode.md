# 基于 OpenCV 的垃圾智能分类系统
## Claude Code 项目实施文档

## 1. 项目定位

### 1.1 项目名称

基于 OpenCV 的垃圾智能分类系统

### 1.2 项目目标

本项目旨在实现一个可训练、可推理、可交互演示的垃圾智能分类系统。系统主要面向毕业设计场景，既要能支撑论文中的技术实现与实验验证，也要能形成一个结构清晰、可持续迭代的项目工程，便于后续由 Claude Code 辅助完成代码生成、模块开发和功能完善。

系统需要具备以下能力：

1. 支持垃圾图像四分类任务。
2. 支持静态图片识别。
3. 支持摄像头实时识别。
4. 支持数据预处理、训练、验证、测试与模型导出。
5. 支持图形界面或轻量 Web 界面演示。
6. 代码结构清晰，适合逐步扩展为论文实验平台。

### 1.3 项目边界

本项目当前聚焦于“图像分类系统”而不是“多目标检测系统”。

当前阶段默认分类标签为以下四类：

1. 可回收物
2. 厨余垃圾
3. 有害垃圾
4. 其他垃圾

当前阶段不强制要求：

1. 垃圾目标检测框选
2. 云端部署
3. 移动端原生 App
4. 嵌入式硬件部署
5. 物联网平台对接

这些内容可以作为后续扩展方向预留接口。

## 3. 技术路线概览

### 3.1 总体技术路线

本项目采用“OpenCV 图像处理 + 深度学习分类模型 + 交互演示系统”的总体技术路线。

技术流程如下：

1. 收集和整理垃圾图像数据集
2. 使用 OpenCV 完成图像清洗、裁剪、缩放、颜色空间转换和增强
3. 训练图像分类模型
4. 对模型进行验证与测试
5. 封装推理接口
6. 使用 OpenCV 完成图片读取、视频流处理、结果叠加显示
7. 构建图形界面或轻量前端演示系统

### 3.2 项目核心模块

建议将项目拆分为以下模块：

1. `config`：配置管理
2. `dataset`：数据集读取与标签管理
3. `opencv_pipeline`：OpenCV 图像处理流程
4. `models`：模型定义与模型工厂
5. `train`：训练流程
6. `eval`：验证与测试流程
7. `infer`：推理流程
8. `ui`：图形界面或 Web 界面
9. `utils`：日志、指标、路径、可视化等工具
10. `experiments`：实验记录和结果输出

## 4. 神经网络模型方案

### 4.1 说明

模型不需要和论文初稿完全一致，当前阶段可先设为“候选模型并行比较”，后续根据实验结果最终确定主模型。

### 4.2 推荐候选模型

建议优先预选以下分类模型：

1. MobileNetV3-Small
2. EfficientNet-B0
3. ResNet18
4. ShuffleNetV2

### 4.3 模型选择策略

建议将模型选择分为两个层次：

1. 基线模型
2. 主力模型

建议方案如下：

1. `ResNet18` 作为准确率基线模型（实测 93.54%）
2. `MobileNetV3-Small` 作为轻量化对比模型（实测 92.30%）
3. `EfficientNet-B0` 作为最终主力模型（实测 94.47%，最优）
4. `ShuffleNetV2` 作为实时性候选模型（实测 93.44%）

### 4.4 模型最终确定逻辑

最终主模型应根据以下指标综合决定：

1. Accuracy
2. Precision
3. Recall
4. F1-score
5. 参数量
6. 推理速度
7. 摄像头实时识别帧率

如果毕业设计重点偏“工程实现”，建议最终主模型优先选择轻量模型。
如果毕业设计重点偏“实验对比”，建议保留 3 到 4 个模型对照实验。

## 5. OpenCV 板块的重点工作

### 5.1 OpenCV 在本项目中的定位

OpenCV 不是辅助模块，而是本项目的核心工程模块之一。论文题目中已经明确突出 OpenCV，因此在项目实施文档中必须体现 OpenCV 的独立工作量和关键作用。

### 5.2 OpenCV 主要工作内容

OpenCV 在本项目中至少承担以下工作：

1. 图像读取与格式兼容处理
2. 图像尺寸统一
3. BGR 与 RGB 颜色空间转换
4. 图像归一化前处理
5. 亮度、对比度和色彩增强
6. 高斯模糊、去噪等预处理
7. 随机裁剪、旋转、翻转等数据增强
8. 摄像头实时视频流采集
9. 视频帧逐帧处理
10. 识别结果文字叠加显示
11. 图像保存与中间结果可视化
12. 数据集预处理脚本开发

### 5.3 OpenCV 单独模块设计建议

建议单独建立如下目录：

```text
opencv_pipeline/
  __init__.py
  io.py
  preprocess.py
  augment.py
  camera.py
  draw.py
  dataset_tools.py
```

各模块职责建议如下：

1. `io.py`：图片读取、格式转换、路径加载
2. `preprocess.py`：Resize、Normalize、颜色空间转换、去噪
3. `augment.py`：翻转、旋转、亮度扰动、裁剪、噪声增强
4. `camera.py`：摄像头读取、视频流采集、帧控制
5. `draw.py`：分类结果、提示文字、概率信息叠加到图像
6. `dataset_tools.py`：数据集清洗、批量预处理、可视化抽样

### 5.4 OpenCV 板块应体现的工作量

为了在毕业设计中突出 OpenCV 的工作量，项目开发时建议明确保留以下可展示成果：

1. 数据集批量预处理脚本
2. 数据增强前后对比图
3. 不同预处理策略的效果对比
4. 摄像头实时识别演示脚本
5. 分类结果叠加界面
6. 可视化样本检查工具
7. 图像质量异常检测工具

这些成果不仅有助于代码实现，也有助于论文中撰写“系统实现”和“实验分析”章节。

## 9. 推理与演示系统设计

### 9.1 单图推理

单图推理功能要求如下：

1. 输入图片路径
2. 自动执行预处理
3. 输出预测类别
4. 输出类别概率
5. 保存结果图像

### 9.2 摄像头实时识别

实时识别功能要求如下：

1. 调用默认摄像头
2. 按帧读取图像
3. 对每一帧执行 OpenCV 预处理
4. 完成模型推理
5. 在画面上叠加类别与概率
6. 支持退出控制
7. 统计 FPS

### 9.3 界面层方案

界面层可优先考虑以下两种方案之一：

1. `Tkinter/PySide6` 桌面 GUI
2. `FastAPI + HTML` Web UI（已采用）

实际实施结果：采用 FastAPI + HTML 方案，实现 4-Tab Web 系统（首页/智能识别/批量识别/OpenCV演示），
支持模型对比、批量识别、OpenCV 预处理管线可视化、识别历史等功能，通过浏览器 localhost:8080 访问。

## 13. 当前建议结论

当前项目建议采用如下策略启动：

1. 先搭建以 `OpenCV + EfficientNet-B0` 为核心的最小可运行版本（实测最优 94.47%）
2. 同时保留 `ResNet18`、`MobileNetV3-Small`、`ShuffleNetV2` 作为对比模型
3. 优先完成 OpenCV 预处理、摄像头视频流、结果可视化三大模块
4. 训练与界面两条线并行推进
5. 保持代码、实验和论文材料同步整理

如果后续需要扩展，优先扩展方向建议为：

1. 错分样本分析工具
2. 数据增强策略对比实验
3. 实时识别 FPS 优化
4. 多模型对照实验
