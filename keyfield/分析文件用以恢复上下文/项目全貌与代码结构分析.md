## 3. 核心模块功能说明

### 3.1 配置系统 (`configs/` + `src/config.py`)

- `base.yaml`：全局基础配置（项目名、seed、数据路径、模型参数、标签映射、输出目录）
- `train.yaml`：训练覆盖配置（epochs=50, batch_size=128, lr=0.001, AdamW, CosineAnnealingLR, 两阶段训练, OpenCV预处理开关, 数据增强参数）
- `config.py`：将 YAML 递归合并后转为嵌套 dataclass（ProjectConfig），提供 `seed_everything()` 固定随机种子

### 3.2 数据集 (`src/dataset.py`)

- `LABEL_MAP`：recyclable=0, kitchen=1, hazardous=2, other=3
- `WasteDataset`：PyTorch Dataset，流程：OpenCV读BGR → 可选OpenCV预处理 → BGR→RGB → PIL → torchvision transform → tensor
- `OpenCVPreprocess`：可序列化预处理类（去噪+CLAHE），支持 Windows 多进程 DataLoader
- `create_dataloaders()`：根据配置构建 train/val/test DataLoader

### 3.3 模型 (`src/models/`)

- 统一注册机制：`@register_model("模型名")` 装饰器注册到 `MODEL_REGISTRY`
- `build_model(config)` 根据配置名实例化模型
- 每个模型均支持：预训练权重加载、backbone冻结/解冻、替换分类头为4类输出

| 模型 | 参数量 | 分类头位置 |
|------|--------|-----------|
| MobileNetV3-Small | ~2.5M | classifier[3] |
| ResNet18 | ~11.7M | fc |
| EfficientNet-B0 | ~5.3M | classifier[1] |
| ShuffleNetV2 | ~2.3M | fc |

### 3.4 训练引擎 (`src/train.py`)

- **两阶段训练**：阶段一（1-15 epoch）冻结backbone只训练分类头；阶段二（16-50 epoch）解冻backbone，学习率×0.1
- EarlyStopping（patience=10, min_delta=0.001）
- 自动保存最佳模型权重（监控 val_accuracy）
- 输出：训练日志CSV、Loss/Accuracy/LR三合一训练曲线图、配置快照YAML

### 3.5 评估引擎 (`src/evaluate.py`)

- 计算指标：Overall Accuracy、Macro/Per-class Precision/Recall/F1
- 混淆矩阵热力图（行归一化，显示数量+百分比）
- 错分样例导出（PIL渲染中文标签：真实类别/预测类别）
- Markdown评估报告自动生成

### 3.6 推理引擎 (`src/infer.py`)

- `WasteClassifier`：加载checkpoint → 构建模型 → 推理
- `predict_image()`：单图推理，返回类别名、类别ID、置信度、四类概率分布
- `predict_batch()`：批量推理
- `predict_and_annotate()`：推理+绘制标注图（类别+概率条形图）

### 3.7 OpenCV管线 (`src/opencv_pipeline/`)

- **preprocess.py**：resize、normalize、颜色空间转换(BGR↔RGB/HSV/LAB)、去噪(高斯/中值/双边)、CLAHE(LAB空间L通道)、亮度/对比度调整、Canny/Sobel/Laplacian边缘检测、PreprocessPipeline可配置管线
- **augment.py**：随机水平/垂直翻转、随机旋转(±15°)、随机亮度扰动、随机裁剪、高斯噪声、AugmentationPipeline可配置增强管线
- **camera.py**：CameraStream(摄像头管理+分辨率设置)、FPSCounter(滑动窗口FPS计算)
- **draw.py**：draw_result(类别+置信度)、draw_fps、draw_probability_bar(四类概率条形图)、PIL渲染中文避免乱码

### 3.8 Web界面 (`ui/`)

**后端 `app.py`**：
- `ModelManager`：多模型懒加载与切换管理
- 6个API接口：
  - `POST /api/predict`：单图分类（返回类别/置信度/概率/标注图base64/投放建议/科普信息）
  - `GET /api/models`：列出可用模型
  - `POST /api/switch-model`：切换当前模型
  - `POST /api/compare`：四模型同图对比推理（含推理耗时）
  - `POST /api/batch-predict`：批量识别（含类别统计、平均置信度）
  - `POST /api/opencv-pipeline`：OpenCV预处理管线可视化（原图→去噪→CLAHE→resize）
- 四大类科普信息：颜色、垃圾桶颜色、描述、示例、投放建议

**前端（多标签页）**：
- 首页：系统概览、数据集规模、四类垃圾颜色编码、快速上传入口
- 智能识别页：单图识别（标注图+概率+建议）+ 四模型对比（四宫格+表格）
- 批量识别页：多图上传 + ECharts类别统计图
- OpenCV演示页：预处理管线逐步可视化
- 历史记录：localStorage保存最近20条

## 4. 数据集详情

### 4.1 来源与规模

- 原始数据：245个细分类别文件夹，命名格式"大类_小类"（如"厨余垃圾_苹果"）
- 总图片数：80012 张

### 4.2 四分类映射

| 类别 | 英文 | 子类数 | 图片数 | 占比 |
|------|------|--------|--------|------|
| 可回收物 | recyclable | 136 | 49576 | 62.0% |
| 厨余垃圾 | kitchen | 48 | 20483 | 25.6% |
| 其他垃圾 | other | 47 | 5151 | 6.4% |
| 有害垃圾 | hazardous | 14 | 4802 | 6.0% |

### 4.3 数据划分（8:1:1分层抽样）

| 类别 | 总数 | 训练集 | 验证集 | 测试集 |
|------|-----:|------:|------:|------:|
| 可回收物 | 49576 | 39660 | 4957 | 4959 |
| 厨余垃圾 | 20483 | 16386 | 2048 | 2049 |
| 有害垃圾 | 4802 | 3841 | 480 | 481 |
| 其他垃圾 | 5151 | 4120 | 515 | 516 |
| **合计** | **80012** | **64007** | **8000** | **8005** |

### 4.4 数据特点

- **类别极不均衡**：可回收物占62%，有害垃圾+其他垃圾不足13%
- **子类丰富**：可回收物包含136种（不锈钢制品、书、充电宝、塑料瓶等），厨余垃圾48种（苹果、蛋糕、蔬菜等）
- **背景复杂**：桌面纹理、塑料袋、容器边缘、阴影与高光反射等干扰

| 参数 | 数值 | 说明 |
|------|-----|------|
| 输入尺寸 | 224×224 | 所有模型统一 |
| 批量大小 | 128 | |
| 训练轮次 | 50 | |
| 初始学习率 | 0.001 | |
| 优化器 | AdamW | weight_decay=1e-4 |
| 学习率调度 | CosineAnnealingLR | |
| 阶段一轮次 | 15 | 冻结backbone |
| 阶段二学习率因子 | 0.1 | 解冻后lr×0.1 |
| EarlyStopping | patience=10 | 监控val_accuracy |
| 损失函数 | CrossEntropyLoss | |
| 数据加载线程 | 8 | |

### OpenCV预处理（训练+推理均启用）

- 高斯去噪：ksize=3
- CLAHE：clipLimit=2.0, tileGridSize=(8,8)，在LAB空间L通道操作

### 数据增强（训练时）

- 随机水平翻转(p=0.5)、随机垂直翻转(p=0.5)
- 随机旋转(±15°)
- ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2)
- RandomErasing(p=0.2)

## 6. 实验结果

### 6.1 第二轮正式训练结果（含OpenCV预处理，round2/）

| 模型 | Accuracy | Macro Precision | Macro Recall | Macro F1 | 权重/MB |
|------|---------:|----------------:|-------------:|---------:|--------:|
| **EfficientNet-B0** | **94.47%** | **91.64%** | **87.70%** | **89.54%** | 46.39 |
| ResNet18 | 93.57% | 90.20% | 85.75% | 87.81% | 128.06 |
| ShuffleNetV2 | 93.39% | 89.41% | 85.83% | 87.52% | 14.70 |
| MobileNetV3-Small | 92.38% | 88.15% | 82.66% | 85.11% | 17.67 |

### 6.2 各类别F1值对比

| 类别 | MobileNetV3-Small | ResNet18 | EfficientNet-B0 | ShuffleNetV2 |
|------|------------------:|---------:|----------------:|-------------:|
| 可回收物 | 94.47% | 95.27% | **96.00%** | 95.14% |
| 厨余垃圾 | 94.80% | 95.58% | **96.03%** | 95.53% |
| 有害垃圾 | 82.05% | 85.01% | **87.67%** | 83.73% |
| 其他垃圾 | 69.12% | 75.39% | **78.46%** | 75.67% |

### 6.3 推理耗时（RTX 3060 Laptop，5次平均）

| 模型 | 耗时/ms |
|------|--------:|
| ResNet18 | 8.2 |
| MobileNetV3-Small | 11.9 |
| ShuffleNetV2 | 12.8 |
| EfficientNet-B0 | 15.4 |

### 6.4 OpenCV预处理对比

- 离线预处理将每epoch训练时间从 ~400s 降至 ~130s（提升约3倍）
- EfficientNet-B0：第一轮 94.43% → 第二轮 94.47%（+OpenCV预处理，精度小幅提升）
- 对轻量模型（MobileNetV3-Small）精度影响不大，但训练效率显著提升

### 6.5 训练过程关键节点（EfficientNet-B0）

- Epoch 1：Train Acc 80.70%, Val Acc 84.39%
- Epoch 15（阶段一结束）：Val Acc 86.80%
- Epoch 16（解冻微调开始）：Val Acc 跃升至 91.33%
- Epoch 41（最佳）：Val Acc 94.94%
- Epoch 50（最终）：Train Acc 99.40%, Val Acc 94.73%

## 7. 主要误差分析

- **其他垃圾误判最多**：与可回收物材质交叉（如PE塑料袋被误判为可回收物）
- **有害垃圾次之**：样本量少（仅4802张），指甲油瓶外观与化妆品瓶相似
- **可回收物/厨余垃圾**：识别稳定，F1 > 94%
