## 三、逐模块详细对比

### 3.1 数据集

| 维度 | 参考项目 | 毕设需求 | 差距与对策 |
|------|---------|---------|----------|
| **数据量** | 80,012张/245类 | 需四分类数据集 | **可直接复用**，将245类映射到4大类即可 |
| **目录结构** | `trash_jpg/{大类}_{小类}/` | `data/{raw,cleaned,processed,train,val,test,meta}/` | 需编写数据整理脚本，将原始数据重新组织 |
| **标签体系** | 中文 `其他垃圾_PE塑料袋` | `0:recyclable, 1:kitchen, 2:hazardous, 3:other` | 需建立映射表：从245细分类名前缀提取大类 |
| **数据划分** | Keras 自动划分（比例不一致） | 标准 train/val/test 独立划分 | 需重写划分脚本，参考 data_spilit.py 但需规范化 |
| **数据清洗** | jpeg2jpg.py, remove_wrong_image.py | 需系统化清洗流程 | 可复用清洗逻辑，但需整合到 opencv_pipeline 模块 |

**映射关系（245类→4大类）**：
- `其他垃圾_*` → other（47个子类，~15,000张）
- `厨余垃圾_*` → kitchen（49个子类，~15,000张）
- `可回收物_*` → recyclable（136个子类，~45,000张）
- `有害垃圾_*` → hazardous（13个子类，~5,000张）

**关键问题**：类别极度不均衡——可回收物（136类）远多于有害垃圾（13类），需要在训练时处理。

### 3.2 深度学习框架

| 维度 | 参考项目 | 毕设需求 |
|------|---------|---------|
| **框架** | TensorFlow 2.3 / Keras | 开题报告提 PyTorch，实施文档不限定 |
| **Python** | 3.7 | 3.12（已就绪） |
| **模型保存** | .h5 格式 | .pth 或 SavedModel |
| **API** | 大量 deprecated API | 需全部更新 |

**建议**：迁移到 **PyTorch**，理由：
1. 开题报告中已提到 PyTorch
2. Python 3.12 对 PyTorch 支持良好（PyTorch 2.x）
3. 参考项目中的 Keras 模型代码无法直接运行，不如重写
4. PyTorch 的数据增强、训练循环更灵活

### 3.3 模型架构

| 参考项目模型 | 毕设候选模型 | 对应关系 |
|------------|------------|---------|
| 自定义 CNN（2层Conv） | - | 不推荐复用，结构过于简单 |
| MobileNetV2（迁移学习） | MobileNetV3-Small（主力） | **可参考迁移学习思路**，但需换成 V3-Small |

## 七、毕设关键技术路线确认

根据开题报告和实施文档，毕设的核心技术路线为：

1. **数据**：TrashNet/TACO 等公开数据集 + 当前参考项目的80,012张图片
2. **预处理**：OpenCV（Resize, Normalize, CLAHE, 去噪, 颜色空间转换）
3. **增强**：OpenCV（翻转, 旋转, 亮度扰动, 裁剪, 噪声增强）
4. **模型**：MobileNetV3-Small 为主力，ResNet18/EfficientNet-B0/ShuffleNetV2 为对比
5. **优化**：迁移学习 + 注意力机制 + 学习率调度 + 标签平滑
6. **推理**：单图推理 + 摄像头实时识别（OpenCV VideoCapture）
7. **界面**：Tkinter 或 PySide6 桌面 GUI
8. **评估**：Accuracy/Precision/Recall/F1/混淆矩阵/参数量/推理速度/FPS
