## 一、项目概述

本项目是一个基于 **TensorFlow 2.3** 的图像分类系统，用于对中国城市生活垃圾进行智能分类。系统包含约 **80,012 张图片**，覆盖 **245 个细分类别**，归入 4 大垃圾分类（厨余垃圾、可回收物、其他垃圾、有害垃圾）。

原始作者：肆十二 / dejahu（B站UP主）
原始教程视频：https://www.bilibili.com/video/BV1rX4y1A7N8/

## 三、模型架构分析

### 3.1 自定义 CNN 模型（train_cnn.py）

| 项目 | 详情 |
|------|------|
| **架构** | 自定义 2 层卷积网络 |
| **输入** | 224x224x3 RGB 图片 |
| **层结构** | Rescaling(1/255) → Conv2D(32,3x3,relu) → MaxPool(2x2) → Conv2D(64,3x3,relu) → MaxPool(2x2) → Flatten → Dense(128,relu) → Dense(245,softmax) |
| **参数量** | ~23,938,997（全部可训练） |
| **优化器** | Adam |
| **损失函数** | categorical_crossentropy |
| **Epochs** | 30 |
| **验证集准确率** | **67.5%** |
| **模型文件** | cnn_245_epoch30.h5 |

**问题分析**：
- 网络结构过于简单，仅 2 层卷积，对 245 类分类任务能力不足
- Flatten 后直接接 Dense(128)，参数量巨大（~23.8M）且容易过拟合
- 没有 Dropout、BatchNorm 等正则化手段
- 67.5% 的准确率偏低，模型泛化能力有限

### 3.2 MobileNetV2 迁移学习模型（train_mobilenet.py）

| 项目 | 详情 |
|------|------|
| **架构** | MobileNetV2（ImageNet 预训练）+ 自定义分类头 |
| **输入** | 224x224x3 RGB 图片 |
| **层结构** | Rescaling(1/127.5,offset=-1) → MobileNetV2(frozen) → GlobalAvgPool2D → Dense(245,softmax) |
| **参数量** | ~2,571,829（可训练 313,845） |
| **优化器** | Adam |
| **损失函数** | categorical_crossentropy |
| **Epochs** | 30 |
| **验证集准确率** | **82.9%** |
| **模型文件** | mobilenet_245_epoch30.h5 |

**问题分析**：
- 基座模型完全冻结（base_model.trainable=False），仅训练分类头，可能限制精度上限
- 没有使用数据增强（Data Augmentation）
- 没有使用学习率调度、EarlyStopping、ModelCheckpoint 等回调
- 82.9% 的准确率尚可但仍有提升空间

### 3.3 ResNet50 模型（utils/train_resnet.py - 未正式使用）

- 代码存在于 utils 目录中，结构同 MobileNet 脚本
- 使用 ResNet50 作为 base model，同样冻结基座
- **未被正式纳入项目流程**，模型未保存到 models 目录

## 四、245 类别分类体系

4 大类及其子类数量：

| 大类 | 子类数量 | 示例 |

| 其他垃圾 | 47 | PE塑料袋、口罩、烟蒂、打火机、餐巾纸... |

**类别分布不均衡**：可回收物（136类）远多于有害垃圾（13类），这在训练中可能导致类别偏斜问题。

## 五、GUI 桌面应用分析（window_trash.py）

| 项目 | 详情 |
|------|------|
| **框架** | PyQt5 (QTabWidget) |
| **窗口尺寸** | 900x700 |
| **主页面** | 左侧图片展示 + 右侧识别结果 |
| **关于页面** | 作者信息展示 |
| **功能** | 上传图片 → 预测 → 显示垃圾大类+物品名称+分类说明 |

**问题分析**：
- class_names 列表硬编码在 3 个文件中（train_cnn.py, test_model.py, window_trash.py），共重复 3 次
- 使用 shutil.copy 做临时文件处理，通过固定的 images/t1.jpg、images/t2.jpg 等路径，多线程不安全
- 图片预测时没有做异常处理（空图片、非 RGB 图片等）
- 界面中文使用"楷体"字体，非跨平台通用
- 路径全部使用相对路径，只能在 code 目录下运行

### 7.3 关键兼容性问题

1. **TensorFlow 2.3.0 不支持 Python 3.12**：TF 2.3 最高支持 Python 3.8。当前 Python 3.12 需要至少 TensorFlow 2.16+
2. **已弃用的 API**：
   - `tf.keras.preprocessing.image_dataset_from_directory` → 应改为 `tf.keras.utils.image_dataset_from_directory`
   - `tf.keras.layers.experimental.preprocessing.Rescaling` → 应改为 `tf.keras.layers.Rescaling`
3. **模型格式**：`.h5` 格式在较新版本 TensorFlow 中仍可用但已非默认推荐格式（推荐 SavedModel）
4. **数据路径硬编码**：所有训练脚本中数据路径写死为 `F:/datas/tmp/data/tttt/trash_jpg`，需要统一修改

## 九、迁移重构建议（概要）

### 9.1 框架升级方向

| 现状 | 建议方向 |
|------|---------|
| TensorFlow 2.3 + Keras | 升级至 TensorFlow 2.16+ 或迁移至 PyTorch |
| Python 3.7 | Python 3.12（已就绪） |
| .h5 模型格式 | SavedModel 或 .keras 格式 |
| 硬编码参数 | YAML/JSON 配置文件 |
| 重复代码 | 提取公共模块（config.py, dataset.py, model.py） |

### 9.2 推荐重构后的项目结构

```
project/
├── configs/
│   └── config.yaml              # 统一超参数配置
├── data/
│   └── trash_jpg/               # 数据集（245个子文件夹）
├── src/
│   ├── __init__.py
│   ├── config.py                # 配置加载模块
│   ├── dataset.py               # 数据加载与预处理（统一 data_load）
│   ├── models/
│   │   ├── cnn.py               # CNN 模型定义
│   │   ├── mobilenet.py         # MobileNet 模型定义
│   │   └── resnet.py            # ResNet 模型定义
│   ├── train.py                 # 统一训练入口
│   ├── evaluate.py              # 统一评估入口
│   └── utils.py                 # 工具函数
├── gui/
│   ├── main_window.py           # PyQt5 主界面
│   └── resources/               # UI 资源文件
├── saved_models/                # 模型保存目录
├── results/                     # 训练结果与可视化
├── requirements.txt
└── README.md
```

### 9.3 重构优先级

1. **P0 - 环境搭建**：安装适配 Python 3.12 的 TensorFlow 版本
2. **P1 - 代码模块化**：抽取公共模块（dataset.py, config.py），消除重复代码
3. **P2 - API 更新**：替换 deprecated API（image_dataset_from_directory, Rescaling）
4. **P3 - 配置外置**：将硬编码的路径和超参数提取到配置文件
5. **P4 - 模型升级**：增加数据增强、学习率调度、微调策略等
6. **P5 - GUI 改造**：解耦界面与模型逻辑，增加异常处理

## 十二、总结

本项目是一个功能完整的垃圾分类图像识别系统，涵盖了从数据预处理、模型训练、模型评估到桌面 GUI 应用的完整流程。但由于原始代码基于 2021 年的 TensorFlow 2.3 环境，存在以下核心问题需要重构：

1. **技术栈过时**：TF 2.3 无法在 Python 3.12 上运行，大量 API 已废弃
2. **代码耦合度高**：配置、数据加载、模型定义、训练逻辑混在一起
3. **重复代码严重**：class_names 和 data_load 函数在多个文件中重复
4. **路径硬编码**：数据路径和模型路径写死为开发者本机路径
5. **缺少工程化实践**：无配置管理、无日志、无异常处理、无测试

这些问题为后续的全面迁移重构提供了明确的改进方向。
