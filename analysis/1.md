   项目"基于OpenCV的垃圾智能分类系统"需要 Web 端 GUI 界面用于论文答辩演示。                                                                                     
     使用 FastAPI 后端 + HTML/CSS/JS 前端，先用本地 2 epoch 权重开发，云端训练完成后替换权重文件即可。

     需求确认

     - 图片上传分类（核心功能）
     - 四大类科普信息（联动显示）
     - 模型切换（4个模型下拉选择）
     - 左右分栏布局，简洁现代风
     - 四大类颜色区分（蓝/绿/红/灰）
     - 仅 localhost:8080，一键 start.bat 启动

     技术架构

     浏览器 (localhost:8080)
         |
         v
     FastAPI (ui/app.py)
         ├── GET  /                  → 返回 index.html
         ├── POST /api/predict       → 图片上传 → 推理 → 返回 JSON 结果
         ├── GET  /api/models        → 返回可用模型列表
         └── POST /api/switch-model  → 切换当前使用的模型
         |
         v
     src/infer.py (WasteClassifier)
         ├── predict_image()         → 返回 {class_name, confidence, probabilities}
         └── 使用 src/opencv_pipeline/draw.py 标注图片

     API 设计

     POST /api/predict

     - 请求: multipart/form-data, 字段 file=图片, model=模型名(可选)
     - 响应 JSON:
     {
       "class_name": "可回收物",
       "class_id": 0,
       "confidence": 0.952,
       "probabilities": {
         "可回收物": 0.952,
         "厨余垃圾": 0.021,
         "有害垃圾": 0.015,
         "其他垃圾": 0.012
       },
       "advice": "请投入蓝色垃圾桶。可回收物包括：纸张、塑料、玻璃、金属、织物等。",
       "image_id": "abc123"
     }

     GET /api/models

     - 响应: 可用模型列表 + 当前选中模型

     GET /api/annotated/{image_id}

     - 响应: 标注后的图片（OpenCV draw 生成的）

     前端页面设计

     左侧面板（60%宽度）

     1. 图片上传区 — 拖拽/点击上传，支持预览
     2. 分类结果卡片 — 类别名（大字）+ 置信度百分比 + 颜色标识
     3. 概率分布柱状图 — 四个类别的水平柱状图，用 Canvas 绘制
     4. 投放建议 — 联动显示该类垃圾的投放说明

     右侧面板（40%宽度）

     1. 模型选择下拉框 — MobileNetV3-Small / ResNet18 / EfficientNet-B0 / ShuffleNetV2
     2. 四大类科普卡片 — 四个可折叠/展开的卡片，显示各类垃圾说明
     3. 系统信息 — 当前模型、参数量、设备等

     科普信息内容

     四大类各准备投放指南：
     - 可回收物：纸张、塑料瓶、玻璃、金属、织物 → 蓝色桶
     - 厨余垃圾：果皮、剩菜、茶叶渣、骨头 → 绿色桶
     - 有害垃圾：电池、灯泡、过期药品、油漆 → 红色桶
     - 其他垃圾：餐巾纸、烟蒂、陶瓷、尘土 → 灰色桶
