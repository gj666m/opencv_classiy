## 四、OpenCV 图像处理模块（核心模块）

### 4.1 模块设计目标
论文题目要求 OpenCV 作为独立核心模块，因此所有图像预处理、增强、可视化均使用 OpenCV 实现。

### 4.2 预处理管线（PreprocessPipeline）

```python
# 配置文件: configs/train.yaml
opencv_preprocess:
  enabled: true
  denoise: true                # 高斯去噪
  denoise_method: "gaussian"   # 去噪方法: gaussian / median / bilateral
  denoise_ksize: 3             # 卷积核大小
  clahe: true                  # 自适应直方图均衡化
  clahe_clip_limit: 2.0        # 对比度限制参数
```

#### 处理流程：
```
原始图片 → BGR读取
  → 高斯去噪 (cv2.GaussianBlur, kernel=3×3)
  → 转LAB颜色空间 (cv2.cvtColor, BGR2LAB)
  → 分离L通道
  → CLAHE均衡化 (cv2.createCLAHE, clipLimit=2.0, gridSize=8×8)
  → 合并LAB通道
  → 转回BGR (cv2.cvtColor, LAB2BGR)
  → 输出处理后的图片
```

#### 各步骤原理：
- **高斯去噪**：使用 3×3 高斯核平滑图像，减少采集噪声和压缩伪影
- **CLAHE**：Contrast Limited Adaptive Histogram Equalization，将图像分块（8×8），对每块做直方图均衡化，clipLimit=2.0 限制对比度增强幅度，避免噪声放大
- **LAB颜色空间**：只在亮度通道(L)做均衡化，保留颜色信息(a, b)不变

### 4.3 数据增强模块（augment.py）

```python
# 训练时数据增强配置
augmentation:
  random_horizontal_flip: true     # 50%概率水平翻转
  random_vertical_flip: true       # 50%概率垂直翻转
  random_rotation: 15              # 随机旋转 ±15°
  color_jitter:
    brightness: 0.2                # 亮度随机变化 ±20%
    contrast: 0.2                  # 对比度随机变化 ±20%
    saturation: 0.2                # 饱和度随机变化 ±20%
  random_erasing: true             # 随机擦除（20%概率）
  gaussian_noise: false            # 高斯噪声（未启用）
```

#### 增强方法说明：
| 方法 | 参数 | 作用 | 对应论文内容 |
|------|------|------|------------|
| 水平翻转 | p=0.5 | 增加视角多样性 | 几何变换增强 |
| 垂直翻转 | p=0.5 | 适应倒置拍摄 | 几何变换增强 |
| 随机旋转 | ±15° | 适应倾斜拍摄 | 几何变换增强 |
| 颜色抖动 | ±20% | 模拟不同光照 | 颜色空间增强 |
| 随机擦除 | p=0.2 | 模拟遮挡 | 正则化增强 |

### 4.4 结果可视化模块（draw.py）

- `draw_result()`：在图片上绘制分类结果文字 + 置信度
- `draw_probability_bar()`：绘制四类概率分布条形图
- 颜色编码：可回收物=蓝色、厨余垃圾=绿色、有害垃圾=红色、其他垃圾=灰色
- 中文文字渲染使用 PIL（替代 cv2.putText 解决中文乱码问题）

### 4.5 摄像头接口模块（camera.py）

- 基于 `cv2.VideoCapture` 实现实时采集
- 支持 FPS 计算和帧率控制
- 为实时识别场景预留接口

## 五、模型设计

### 5.1 模型选型策略

| 模型 | 定位 | 特点 |
|------|------|------|
| **EfficientNet-B0** | **主力模型**（实验最优） | 复合缩放，参数利用率高，94.47% |
| ResNet18 | 经典对比模型 | 残差连接，训练稳定 |
| MobileNetV3-Small | 轻量化对比模型 | 倒残差结构+SE注意力，适合移动端 |
| ShuffleNetV2 | 实时性对比模型 | 通道混洗，推理速度快 |

### 5.2 模型架构详情

#### EfficientNet-B0（主力模型）
```
输入层:  224 × 224 × 3 (RGB)
         ↓
特征提取: EfficientNet-B0 Backbone (ImageNet预训练)
         ├─ MBConv1 (k3, 16ch)     × 1
         ├─ MBConv6 (k3, 24ch)     × 2
         ├─ MBConv6 (k5, 40ch)     × 2
         ├─ MBConv6 (k3, 80ch)     × 3
         ├─ MBConv6 (k5, 112ch)    × 3
         ├─ MBConv6 (k5, 192ch)    × 4
         ├─ MBConv6 (k3, 320ch)    × 1
         └─ Conv1×1 + BN + Swish
         ↓
全局池化: AdaptiveAvgPool2d(1)
         ↓
Dropout: p=0.2
         ↓
分类头:  Linear(1280 → 4)
         ↓
输出层:  Softmax → 4类概率
```
- **总参数量**: ~5.3M
- **分类头替换**: `model.classifier[1] = nn.Linear(1280, 4)`
- **预训练权重**: ImageNet (`efficientnet_b0_rwightman-7f5810bc.pth`)

#### MobileNetV3-Small
```
输入层:  224 × 224 × 3 (RGB)
         ↓
特征提取: MobileNetV3-Small Backbone (ImageNet预训练)
         ├─ 包含倒残差(Inverted Residual) + SE(Squeeze-Excitation)模块
         └─ 激活函数: H-Swish / H-Sigmoid
         ↓
全局池化: AdaptiveAvgPool2d(1)
         ↓
分类头:  Sequential(
           Linear(576 → 1024), H-Swish,
           Dropout(0.2),
           Linear(1024 → 4)
         )
         ↓
输出层:  Softmax → 4类概率
```
- **总参数量**: ~1.5M (可训练: 0.59M)
- **分类头替换**: `model.classifier[3] = nn.Linear(1024, 4)`

#### ResNet18
```
输入层:  224 × 224 × 3 (RGB)
         ↓
特征提取: ResNet18 Backbone (ImageNet预训练)
         ├─ Conv1(7×7, 64, stride=2) + MaxPool(3×3)
         ├─ Layer1: 2 × BasicBlock(64ch)
         ├─ Layer2: 2 × BasicBlock(128ch)
         ├─ Layer3: 2 × BasicBlock(256ch)
         └─ Layer4: 2 × BasicBlock(512ch)
         ↓
全局池化: AdaptiveAvgPool2d(1)
         ↓
分类头:  Linear(512 → 4)
         ↓
输出层:  Softmax → 4类概率
```
- **总参数量**: ~11.7M
- **分类头替换**: `model.fc = nn.Linear(512, 4)`

#### ShuffleNetV2
```
输入层:  224 × 224 × 3 (RGB)
         ↓
特征提取: ShuffleNetV2 Backbone (ImageNet预训练)
         ├─ Conv1(3×3, 24ch) + MaxPool
         ├─ Stage2: 4 × ShuffleUnit(58ch)
         ├─ Stage3: 8 × ShuffleUnit(116ch)
         └─ Stage4: 4 × ShuffleUnit(232ch)
         ↓
全局池化: AdaptiveAvgPool2d(1)
         ↓
分类头:  Linear(1024 → 4)
         ↓
输出层:  Softmax → 4类概率
```
- **总参数量**: ~2.3M
- **分类头替换**: `model.fc = nn.Linear(1024, 4)`

### 5.3 模型注册机制（base.py）

采用工厂模式 + 装饰器注册，统一模型管理：
```python
# 注册模型
@ModelRegistry.register("mobilenet_v3_small")
class MobileNetV3Small(BaseModel):

# 调用模型
model = ModelRegistry.create("mobilenet_v3_small", num_classes=4, pretrained=True)
```

所有模型继承 `BaseModel` 基类，统一接口：`forward()`, `freeze_backbone()`, `unfreeze_backbone()`, `get_param_count()`

### 5.4 迁移学习策略

```
阶段一 (Epoch 1-15): 冻结 Backbone 权重
  → 只训练分类头 (Linear 层)
  → 学习率: 0.001
  → 目的: 快速适应新任务，避免破坏预训练特征

阶段二 (Epoch 16-50): 解冻 Backbone
  → 全网络微调
  → 学习率降为 0.0001 (stage2_lr_factor=0.1)
  → 目的: 精细调整底层特征，提升整体精度
```

### 6.1 超参数总表

| 参数 | 值 | 说明 |

### 6.2 训练流程

```
1. 加载配置 (configs/train.yaml + configs/base.yaml)
2. 设置随机种子 (seed=42)
3. 创建 DataLoader (train + val)
4. 构建模型 (预训练权重 + 替换分类头)
5. 冻结 Backbone
6. 开始训练循环:
   for epoch in 1..50:
     if epoch == 16:
       解冻 Backbone
       学习率 × 0.1
     训练一个 epoch → 记录 train_loss, train_acc
     验证一个 epoch → 记录 val_loss, val_acc
     如果 val_acc 提升 → 保存 best_model.pth
     如果连续 10 epoch 无提升 → 早停
7. 保存最终模型
8. 生成训练曲线图 (Loss + Acc + LR 三合一)
```

### 8.2 API 接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 返回主页面 |
| `/api/predict` | POST | 单图识别：上传图片 + 可选模型，返回分类结果 |

## 十、关键代码片段（论文可用）

### 10.1 OpenCV 预处理核心代码

```python
# 高斯去噪
img = cv2.GaussianBlur(img, (3, 3), 0)

# CLAHE 自适应直方图均衡化
lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB)
l, a, b = cv2.split(lab)
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
l = clahe.apply(l)
img = cv2.cvtColor(cv2.merge([l, a, b]), cv2.COLOR_LAB2BGR)
```

### 10.2 模型注册器

```python
class ModelRegistry:
    _models = {}

    @classmethod
    def register(cls, name):
        def decorator(model_cls):
            cls._models[name] = model_cls
            return model_cls
        return decorator

    @classmethod
    def create(cls, name, **kwargs):
        return cls._models[name](**kwargs)
```

### 10.3 两阶段训练

# 阶段一：冻结 backbone

    param.requires_grad = False
